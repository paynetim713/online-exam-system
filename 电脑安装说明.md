# 新电脑安装说明-新手版

这份文档给第一次配置项目的人使用。请按顺序做，不要跳步骤。

## 0. 这个项目需要装什么

必须安装：

- Node.js：运行前端页面。
- Python：运行后端接口。
- Python 依赖：FastAPI、DeepFace、Pillow 等。
- 浏览器：建议使用 Edge 或 Chrome。
- 摄像头：人脸验证必须使用。

人脸识别相关依赖在 `requirements.txt` 里，其中最重要的是：

- `deepface==0.0.99`
- `Pillow==11.3.0`
- `numpy==1.26.4`

注意：DeepFace 第一次运行时可能会自动下载人脸模型文件，所以第一次人脸测试必须联网。

## 1. 安装 Node.js

1. 打开 Node.js 官网：
   <https://nodejs.org/>

2. 下载 LTS 版本。

3. 一直点下一步安装即可。

4. 安装完成后，打开 PowerShell，输入：

   ```powershell
   node -v
   npm -v
   ```

5. 如果能看到版本号，例如：

   ```text
   v22.x.x
   10.x.x
   ```

   就说明 Node.js 安装成功。

## 2. 安装 Python

1. 打开 Python 官网：
   <https://www.python.org/downloads/>

2. 下载 Python 3.12。

3. 安装时一定要勾选：

   ```text
   Add python.exe to PATH
   ```

4. 然后再点 Install Now。

5. 安装完成后，打开 PowerShell，输入：

   ```powershell
   py --version
   ```

6. 如果能看到类似：

   ```text
   Python 3.12.x
   ```

   就说明 Python 安装成功。

## 3. 进入项目文件夹

项目文件夹是：

```text
D:\学习资料\代写\no.3(风险自适应在线考试系统
```

打开 PowerShell，输入：

```powershell
cd "D:\学习资料\代写\no.3(风险自适应在线考试系统"
```

注意：路径外面要加双引号，因为路径里有中文和括号。

## 4. 安装前端依赖

在项目文件夹里执行：

```powershell
npm install
```

等待它安装完成。

如果中间有黄色 warning，一般可以先不用管。

如果出现红色 error，把错误截图发给项目负责人。

## 5. 安装后端依赖

还在同一个项目文件夹里执行：

```powershell
py -m pip install --upgrade pip setuptools wheel
py -m pip install -r requirements.txt
```

这一步可能比较慢，因为会安装 DeepFace、TensorFlow、OpenCV 等人脸识别相关依赖。

如果下载很慢，可以换网络再试。

## 6. 第一次测试 DeepFace 是否能正常导入

在 PowerShell 里执行：

```powershell
py -c "from deepface import DeepFace; print('DeepFace OK')"
```

如果看到：

```text
DeepFace OK
```

说明 DeepFace 基本安装成功。

如果这里报错，后面人脸测试很可能一直是 0。

## 7. 启动项目

在项目文件夹里执行：

```powershell
.\start-local.ps1
```

正常情况下会看到：

```text
Local services started.
Frontend: http://127.0.0.1:5173/
Backend:  http://127.0.0.1:8000/api/health
```

然后打开浏览器访问：

```text
http://127.0.0.1:5173/
```

## 8. 检查后端是否启动成功

浏览器打开：

```text
http://127.0.0.1:8000/api/health
```

如果看到类似：

```json
{"status":"ok","service":"..."}
```

说明后端正常。

如果打不开，说明后端没启动成功。

这时看这个文件：

```text
api-dev.err.log
```

里面会有后端报错信息。

## 9. 浏览器摄像头权限

人脸测试前必须确认浏览器允许摄像头。

1. 打开系统页面：

   ```text
   http://127.0.0.1:5173/
   ```

2. 进入人脸验证页面。

3. 浏览器弹出摄像头权限时，选择允许。

4. 如果没有弹窗，点浏览器地址栏左边的小图标，检查摄像头权限是否被禁止。

5. Windows 也要允许摄像头：

   ```text
   设置 -> 隐私和安全性 -> 摄像头
   ```

   确认摄像头权限是打开的。

## 10. 第一次人脸测试为什么可能很慢

第一次做人脸测试时，DeepFace 可能会下载 Facenet 模型文件。

所以第一次测试时：

- 必须联网。
- 可能会卡一会儿。
- 不要马上关闭窗口。

如果模型下载失败，人脸测试可能会一直显示 0。

## 11. 如果人脸测试一直是 0，按这个顺序排查

### 第一步：确认摄像头画面正常

页面里必须能看到自己的人脸。

请保证：

- 脸在画面中间。
- 脸不要太小。
- 光线要亮。
- 不要戴口罩。
- 不要用很暗的房间。
- 摄像头不要被其他软件占用。

### 第二步：确认后端接口通了

按 F12 打开浏览器开发者工具。

进入 Network，找到：

```text
face-verify
```

如果状态码是 `200`，说明接口通了。

但是 `200` 不代表人脸识别成功，还要点开 Response 看：

```json
"similarity": 0
```

如果这里是 0，说明后端没有识别到有效人脸，或者 DeepFace 出错了。

### 第三步：看后端错误日志

打开项目里的：

```text
api-dev.err.log
```

如果里面出现 DeepFace、TensorFlow、OpenCV、model download 相关错误，说明是人脸识别环境没有装好，或者模型没有下载成功。

### 第四步：重新安装 Python 依赖

在项目文件夹里执行：

```powershell
py -m pip install --upgrade pip setuptools wheel
py -m pip install -r requirements.txt --force-reinstall
```

然后重新启动项目：

```powershell
.\start-local.ps1
```

### 第五步：换 Edge 或 Chrome 再试

不要用微信内置浏览器打开。

建议使用：

- Microsoft Edge
- Google Chrome

## 13. 最重要的一句话

如果换电脑后人脸分数一直是 0，优先检查：

1. Python 依赖有没有完整安装。
2. DeepFace 能不能正常导入。
3. 第一次运行时有没有联网下载模型。
4. 浏览器有没有摄像头权限。
5. 摄像头画面里脸是否清楚、居中、够亮。

## 14. 页面加载很久怎么办

如果网速显示有 `3MB/s`，但项目还是加载很久，不一定是普通网页慢。

这个项目第一次运行人脸识别时，DeepFace / TensorFlow 可能会：

- 下载人脸识别模型文件。
- 初始化 TensorFlow。
- 加载 OpenCV / NumPy 等依赖。

所以第一次打开或第一次做人脸验证时可能比较慢。

重点检查两个日志文件：

```text
api-dev.err.log
vite-dev.err.err
```

如果 `api-dev.err.log` 里一直在下载模型，等它完成。

如果里面出现报错，就截图发给项目负责人。

## 15. 执行 start-local.ps1 后，刷新网页后端自动关闭怎么办

正常情况下，刷新网页不会让 Python 后端自动关闭。

如果刷新网页后 `http://127.0.0.1:8000/api/health` 打不开，说明 Python 后端进程崩了。

请按下面步骤排查。

### 第一步：重新启动

在项目目录执行：

```powershell
.\start-local.ps1
```

然后立刻打开：

```text
http://127.0.0.1:8000/api/health
```

如果一开始能打开，刷新网页后打不开，说明后端是在处理请求时崩溃。

### 第二步：查看后端错误日志

打开项目目录里的：

```text
api-dev.err.log
```

把里面最后 30 行截图发给项目负责人。

常见原因：

- Python 版本不兼容。
- DeepFace / TensorFlow 没装完整。
- TensorFlow 模型下载失败。
- OpenCV 或 NumPy 原生库崩溃。
- 杀毒软件拦截 Python。
- 电脑 CPU / 系统环境不兼容当前 TensorFlow 包。

### 第三步：检查 Python 版本

PowerShell 执行：

```powershell
py --version
```

建议使用：

```text
Python 3.12.x
```

不要使用太新的 Python 版本，例如 Python 3.13，因为部分人脸识别依赖可能不稳定。

### 第四步：检查关键依赖能不能导入

在项目目录执行：

```powershell
py -c "import fastapi; print('fastapi ok')"
py -c "import uvicorn; print('uvicorn ok')"
py -c "import numpy; print('numpy ok')"
py -c "import PIL; print('pillow ok')"
py -c "from deepface import DeepFace; print('deepface ok')"
```

哪一条报错，就说明对应依赖没装好。

### 第五步：重新安装后端依赖

如果上一步报错，在项目目录执行：

```powershell
py -m pip install --upgrade pip setuptools wheel
py -m pip install -r requirements.txt --force-reinstall
```

安装完成后，再执行：

```powershell
.\start-local.ps1
```

### 第六步：不要用微信内置浏览器

请使用：

- Microsoft Edge
- Google Chrome

不要直接在微信里打开网页。

### 第七步：如果还是自动关闭

把下面三项发给项目负责人：

1. `api-dev.err.log` 最后 30 行截图。
2. `py --version` 的结果。
3. `py -c "from deepface import DeepFace; print('deepface ok')"` 的结果。

## 16. 网速慢时的页面优化说明

项目已经做了前端分包优化：

- 首次打开页面时，不会一次性加载管理员、监考员、考试主页面的全部代码。
- 登录页会先加载出来。
- 进入某个角色页面时，再加载对应页面代码。

如果只是本地演示，仍然可以用：

```powershell
.\start-local.ps1
```

如果要给别人访问，建议先打包成生产版本：

```powershell
npm run build
npm run preview -- --host 127.0.0.1
```

生产版本通常比开发模式更快。

注意：如果慢的是第一次人脸识别，不是前端页面问题，而是后端 DeepFace / TensorFlow 正在初始化或下载模型。这个只能等模型下载完成，或者提前在网络好的环境中完成一次人脸识别测试。
